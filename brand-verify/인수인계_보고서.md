## 1. 이 작업이 뭔지

월마트/샘스클럽/아마존/이온몰 4개 유통사, 83개 상품유형 베스트셀러 **상위 10위 안에 노출된 브랜드**를 대상으로, NB(내셔널브랜드) 후보 스크리닝용 이슈검증(리콜 이력 / 품질·표시 위반 / 법적·평판 리스크 3개 축)을 진행했습니다. 지시 문서는 `PROMPT_전체_실행지침.md`(총괄) + 파트별 세부 지침 3개(`PROMPT_브랜드검증_실행지침.md`=파트1 월마트/샘스클럽, `PROMPT_이온_이슈검증_실행지침.md`=파트2 이온몰, `PROMPT_아마존_스팟체크_요청.md`=파트3 아마존)였습니다.

## 2. 최종 산출물

**`brand_verify.csv` — 461개 브랜드** (이미 별도 파일로 전달드림)

스키마: `key, 리콜, 품질표시, 법적평판, 5년이내, 비고, 조사일`
- `key`: 브랜드 영문 소문자명 (리포트에서 부분일치로 매칭됨)
- `리콜/품질표시/법적평판`: 통과 / 탈락
- `5년이내`: 탈락 사유 중 최신 건이 2021년 이후면 O, 그 전이면 X, 전부 통과면 -
- `비고`: 근거(연도·사건명)를 가운뎃점(·)으로 구분해 요약

## 4. 작업 방식 / 재개 로직 (다른 컴퓨터에서 이어가실 때 알아야 할 것)

- **진행상황 = `brand_verify.csv` 자체.** 별도 체크포인트 파일 없음. 브랜드 하나 검증할 때마다 `_append.py`로 즉시 append (끝까지 모았다가 한번에 쓰지 않음 — 중간에 끊겨도 그 지점까지 보존).
- **재개 규칙**: 새 브랜드 시작 전 `brand_verify.csv`를 다시 읽어서 해당 key(대소문자 무시, 부분일치)가 이미 있으면 스킵. `_queue.py`가 이 로직을 자동화해줌 (`python3 _queue.py [N]` → 다음 대기 브랜드 N개 출력).
- **판정 기준**: 확인필요 지양 — 뚜렷한 부정근거(공식 리콜 / 소송 패소·합의 / 규제기관 제재) 없으면 기본 통과. 계열사 타 제품 이슈는 "~와는 무관" 명시 후 통과.
- CSV 쓰기는 반드시 `csv.writer(quoting=csv.QUOTE_MINIMAL)` 사용 — 직접 문자열 이어붙이면 비고에 콤마 있을 때 파일 깨짐 (실제로 초기 45개 중 일부가 이 문제로 깨져있었어서 재정비했었음).

## 5. 작업 중 발견한 중요 이슈 (선임께 꼭 알려드려야 할 것)

### 5-1. 크롤러 원본 데이터 품질 문제
- **아마존 크롤링 원본**: `카드 리포트` 시트의 "브랜드(한국명)" 컬럼이 실제 브랜드명이 아니라 상품명이 잘려서 뒤섞인 지저분한 텍스트였음 (예: "발코니 미니 티라미수 스낵"). 반면 이번에 별도로 받은 `아마존_크롤링_원본.xlsx`(브랜드명 원어 컬럼 포함)는 훨씬 깨끗했지만, 그래도 브랜드명 뒤에 "2 pack", "Big", "Fire" 같은 상품 수식어가 붙어 같은 브랜드가 여러 개로 중복 집계되는 문제가 있어 수작업 정제가 필요했음.
- **이온몰 크롤링 원본**: `이온_크롤링_원본.xlsx`는 실제로 **두 개 사이트(일본 AEON + 말레이시아 AEON로 추정)의 크롤링 결과를 하나의 표에 합쳐놓은 것**임. `제품명(원어)` 컬럼의 언어(일본어/영어)로 어느 사이트 결과인지 명확히 구분 가능 (유형별로 한 언어로 통일되거나, 섞인 경우도 순위 중간에서 깔끔하게 끊기는 패턴 — 두 리스트를 이어붙인 형태).
- **크롤러 카테고리 오분류**: 이온몰 데이터 중 `avocado oil`, `chicken tenders`, `french rolls` 등 일부 유형에서 전혀 무관한 상품(스킨케어 니베아, 로레알 헤어제품, 이유식)이 섞여 나옴. 검색엔진 매칭 오류로 추정. **이런 항목은 검증 대상에서 제외 처리함** (실제 상위10 식품 브랜드가 아니라고 판단).
- 아마존 쪽도 `chicken tenders` 유형에 반려동물 간식(Spot Farms, Waggin' Train)이 섞여 들어온 사례가 있어 같은 이유로 제외함.

### 5-2. 겹침(재사용) 전략
- 파트2/3 진행 전, 이미 완료된 397개 브랜드와 대조해서 **겹치는 브랜드는 재검색하지 않고 그대로 재사용**하는 전략을 썼음. 아마존은 미국시장이라 겹침이 꽤 있었고(상위20 기준 137개 중 63개 겹침), 이온몰(일본/동남아 브랜드 위주)은 거의 안 겹쳤음(371개 중 29개만 겹침).
- 이 재사용 전략 덕분에 남은 63개 유형에 대해서도 **먼저 겹침 여부부터 확인하고 신규분만 검색하는 방식으로 진행하면 시간을 크게 아낄 수 있음** — 이어서 하실 때 이 접근을 그대로 쓰시길 권장드림.

### 5-3. 아티팩트/제외 처리
- 크롤러가 만든 잘못된 항목(예: "(2 pack)", "N cans", 브랜드명 잘림)은 `_exclude.txt`에 수동으로 쌓아가며 걸러냄. 이 파일도 함께 전달함 — 다음 작업자가 참고하면 같은 문제를 반복해서 겪지 않음.

## 6. 남은 작업 (다음 담당자가 이어가실 부분)

1. **파트2 이온몰**: 21~83번째 유형(63개) + 상위20 유형 내 말레이시아 사이트분. 앞선 분석 기준 전체 스코프는 약 371개 브랜드 중 342개가 신규 검색 대상으로 추정됨 (이번에 상위20 일본사이트분 16개는 이미 처리함).
2. **파트3 아마존**: 21~83번째 유형(63개). 상위20 기준 신규율(74/137 ≈ 54%)로 추정하면 전체는 신규 약 250개 안팍으로 예상.
3. 두 파트 다 **브랜드 한글명 → 영문명 매핑 확정**이 필요함(`PROMPT_아마존_스팟체크_요청.md` 5절 참고) — 이건 결과 병합 시 저쪽(선임)과 협의 필요한 부분.
