## 2. 출력 파일 (★ 반드시 이 스키마)

`D:\AI 프로젝트\유통사크롤러\brand_verify.csv`에 바로 append (UTF-8 BOM, 콤마 구분):

컬럼 순서 그대로: `key,리콜,품질표시,법적평판,5년이내,비고,조사일`

| 컬럼 | 값 | 설명 |
|---|---|---|
| key | 소문자 브랜드명 | 리포트에서 브랜드명 부분일치로 매칭됨. `sample_products`에서 파악한 **실제 브랜드 풀네임**을 소문자로 (예: `apple & eve`, `lee kum kee`) |
| 리콜 | `통과` / `탈락` | 최근 리콜 이력이 있으면 탈락 |
| 품질표시 | `통과` / `탈락` | 표시위반(원산지·등급 허위표시 등) 소송/제재 있으면 탈락 |
| 법적평판 | `통과` / `탈락` | 그 외 집단소송/규제 벌금/반독점 등 있으면 탈락 |
| 5년이내 | `O` / `X` / `-` | 탈락 사유 중 가장 최근 건이 2021년 이후면 O, 그보다 오래면 X, 전부 통과면 `-` |
| 비고 | 텍스트 | 근거 1~3개를 `·`로 구분해 요약(콤마 대신 가운뎃점 사용 — CSV 깨짐 방지). 통과인데 특이사항 없으면 "검색결과 특이사항 없음" |
| 조사일 | `YYYY-MM-DD` | 오늘 날짜 |

**중요 — 판정 기준(기존 45개 항목과 동일한 기준 유지):**
- 확인필요는 지양한다. **뚜렷한 부정적 근거(공식 리콜, 소송 패소/합의, 규제기관 제재)가 검색으로 안 나오면 기본 `통과`** 처리.
- 해당 브랜드 타품목(계열사 다른 제품)의 이슈는 관련 없으면 비고에 "~와는 무관"이라고 명시하고 통과 처리 (예: Smucker's ↔ 계열사 Jif 살모넬라 리콜은 별개).
- 소소한 이물질 리콜 1건 정도는 리콜=탈락으로 하되 품질표시/법적평판은 별개로 판단(기존 예시 참고: `jif`, `pringles` 행).
- 비고는 반드시 근거(연도·사건명·소송명 등)를 짧게라도 남길 것 — "그냥 통과"만 쓰지 말 것.

**참고용 기존 데이터**: `brand_verify.csv`를 먼저 열어서 이미 채워진 45개 행의 문체·판정 기준을 그대로 따라 하세요 (예: `bertolli`, `carapelli`, `jif` 행).

## 3. 작업 절차 (브랜드 1개당)

1. `_verify_todo.csv`에서 아직 `brand_verify.csv`에 없는 첫 번째 브랜드를 고른다 (아래 4번 재개 규칙 참고).
2. 웹서치로 `<브랜드명> recall`, `<브랜드명> lawsuit`, `<브랜드명> mislabeling` / `false advertising` 등을 검색해 3개 축(리콜/품질표시/법적평판)을 판단한다.
3. 판정 결과를 위 스키마대로 **한 줄** 작성해서 **바로 `brand_verify.csv`에 append**한다 (CSV 쓰기는 파이썬 `csv` 모듈의 `csv.writer(f, quoting=csv.QUOTE_MINIMAL)`을 사용 — 비고에 콤마가 들어가도 안전하게 따옴표 처리되도록. **절대 텍스트 직접 이어붙이기(따옴표 없이) 하지 말 것** — 기존 파일 일부가 이 문제로 이미 깨져있음).
4. 브랜드 1개를 끝낼 때마다 **즉시 파일에 반영**(끝까지 모았다가 한 번에 쓰지 말 것) — 그래야 중간에 멈춰도 여기까지는 저장돼 있음.
5. 다음 브랜드로 넘어가서 반복.

## 4. 재개 규칙 (★ 사용량 한도로 중단됐을 때)

- 매 브랜드 시작 전에 **`brand_verify.csv`를 다시 읽어서 해당 브랜드(key, 대소문자 무시)가 이미 있으면 건너뛴다.**
- 이 규칙 덕분에 세션이 사용량 한도로 중간에 끊겨도, **한도 초기화 후 이 프롬프트 파일을 그대로 다시 붙여넣기만 하면** 이미 끝난 브랜드는 자동으로 스킵되고 안 한 것부터 이어서 진행됩니다. 별도로 "어디까지 했는지" 기록할 필요 없음 — `brand_verify.csv` 자체가 진행상황입니다.
- 한 세션 안에서 여러 브랜드를 처리하다 한도가 임박한 것 같으면, 지금까지 처리한 브랜드가 이미 파일에 append돼 있는지만 확인하고 자연스럽게 종료해도 됨(추가 마무리 작업 불필요).
