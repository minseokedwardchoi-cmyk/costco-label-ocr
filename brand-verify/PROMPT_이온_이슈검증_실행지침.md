# 이온몰(Aeon) 브랜드 이슈검증(리콜/품질표시/법적평판) 실행 지침 — 파트 2

> 이 문서를 그대로 Claude Code에 붙여넣고 실행하면 됩니다.
> 아마존 쪽 파일(`카드 리포트` 시트)에 이미 이 3개 칸이 채워져 있는 방식과 동일한 방식으로,
> **이온몰 파일에도 같은 칸을 새로 만들어서 채우면 됩니다.**
>
> **참고**: `PROMPT_전체_실행지침.md`의 파트 2입니다. 파트 1(월마트/샘스클럽)을 먼저 끝낸 뒤 이어서 진행하세요.

## 0. 배경

우리 프로젝트(월마트/샘스클럽/아마존/이온몰 4개 유통사 83개 유형 베스트셀러 NB후보 스크리닝)에서, 아마존 쪽은 상품마다 **리콜 이력 / 품질·표시 / 법적·평판 리스크 / 비고** 4칸이 이미 채워져 있는데, 이온몰 쪽은 이 칸 자체가 아직 없습니다. 이온몰 담당자가 본인이 갖고 있는 원본 파일(URL 접근 가능)로 직접 검색해서 채워주셔야 합니다 — 저희 쪽에서는 이온몰 URL/캡차 접근이 안 돼서 대신 할 수 없습니다.

## 1. 대상 범위

이온몰 원본 파일에서 **유통사×유형별 상위 10위 안에 든 상품**만 대상입니다(그 아래 순위는 안 해도 됨). 이미 갖고 계신 이온몰 크롤링 결과 파일(유형별 블록, 순위 1~N열 구조)에서 1~10위 열만 보시면 됩니다.

## 2. 채워야 할 칸 (아마존 파일과 동일한 이름/기준으로 맞춰주세요)

각 유형×상품(열) 하나마다 아래 4개 행을 추가:

| 행 이름 | 값 | 기준 |
|---|---|---|
| 리콜 이력 | `통과` / `탈락` | 최근 리콜 이력 있으면 탈락 |
| 품질·표시 | `통과` / `탈락` | 원산지·등급 등 표시위반 소송/제재 있으면 탈락 |
| 법적·평판 리스크 | `통과` / `탈락` | 그 외 집단소송/규제 벌금 등 있으면 탈락 |
| 비고 | 텍스트 | 근거 요약(사건명·연도 등). 특이사항 없으면 "검색결과 특이사항 없음" |

**판정 기준**: 뚜렷한 부정적 근거(공식 리콜, 소송 패소/합의, 규제기관 제재)가 검색으로 안 나오면 기본 `통과`로 처리 (확인필요 지양). 일본 브랜드는 일본어로도 검색해보되(예: `<브랜드명> リコール`, `<브랜드명> 訴訟`), 영어 자료(국제 리콜 데이터베이스 등)도 같이 확인하면 좋습니다.

브랜드명이 일본어 원어라 검색이 애매하면, 해당 브랜드의 **공식 영문 회사명**을 먼저 파악한 뒤(예: 日清オイリオ → Nisshin OilliO Group) 그걸로 검색하세요.

## 3. 저장 형식

기존 이온몰 파일과 같은 구조(유형 블록 안에 행 추가)로 저장하되, **완료 후 저희 쪽 `brand_verify.csv`(월마트/샘스클럽/아마존 공용 브랜드 검증 파일)에도 병합할 수 있도록**, 별도로 아래 형식의 CSV 하나를 추가로 만들어서 같이 보내주시면 저희 쪽에서 자동 병합됩니다:

`key,리콜,품질표시,법적평판,5년이내,비고,조사일`
- `key`: 브랜드 영문명(소문자). 일본 브랜드는 로마자 표기(예: `nisshin oillio`, `j-oil mills`)
- `5년이내`: 탈락 사유가 2021년 이후면 `O`, 그 전이면 `X`, 전부 통과면 `-`
- `조사일`: 작업한 날짜(YYYY-MM-DD)

파이썬 `csv.writer(f, quoting=csv.QUOTE_MINIMAL)`로 저장해주세요(비고에 콤마 있어도 안전하게 처리됨).

## 4. 재개/사용량 관련

- 브랜드(또는 상품) 하나 끝날 때마다 바로바로 파일에 반영해두면, 중간에 세션이 끊겨도 거기까지는 보존됩니다.
- 사용량 한도로 멈추면, 한도 초기화 후 이 프롬프트를 그대로 다시 붙여넣으면 됩니다(이미 채운 칸은 건너뛰고 이어가면 됨).
- 매번 직접 재실행하기 번거로우면, Claude Code의 `schedule`(또는 `/loop`) 기능으로 "몇 시간마다 이 프롬프트 재실행"을 걸어두시는 것도 방법입니다.

## 5. 완료 후

작업하신 파일 + (있으면) 3번의 병합용 CSV를 저희(월마트/샘스클럽 담당)에게 다시 전달해주세요.
